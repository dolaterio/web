Simple and hassle free background tasks

dolater.io runs background jobs where workers are Docker images.

Subscribe to get beta updates

Write the code for your worker

You can use any language! Follow our documentation to write your workers.

Create a Docker image for your worker

Make your worker run in Docker. You can see our examples to get an idea of how to do it.

Create a worker with that image

For us to run your workers we need your docker Image. Can create a public image in hub.docker.com and send us the image name.

Queue jobs to your worker

Use our API to submit jobs to your worker. We'll queue the job and run it as soon as posible!

Get the output of your jobs

When a job is finished you can get its output using our API.

Tutorial

Check our asciify example. This example covers the basic aspects of dolater.io and you'll run a service that will covert any image to ASCII. How useful is that?!

Beta Limits

Although we want to make sure our service is flexible and supports most of the needs, during the beta we need to keep low limits. These are the limitations during the beta:

  • No more than 4 uncompleted jobs per user
  • Memory per job will be limited to 128 Mb
  • All jobs will be killed if they don't finish in 3 minutes.
Drop us a line at admin@dolater.io if you need higher limits during our beta.
